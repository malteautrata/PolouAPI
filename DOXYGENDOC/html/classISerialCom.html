Used to initiate the SerialCom object with port name and baud rate. The function puts the object in the same state as the constructor with transfer parameters. It can be used to change the port name and the baud rate of a SerialCom object.

Parameters
portName: The port name is used to open a serial connection via the port name for the controller specified by the operating system.
baudRate: The baud rate determines the transmission speed at which communication between the PC and controller takes place.

virtual bool ISerialCom::writeSerialCom ( unsigned char  command[],
unsigned short  sizeCommand,
unsigned char *  response,
unsigned short  sizeResponse 
)
pure virtual

This method is used to write commands to and read response from the controller via the open serial connection. If the serial port is not open an exception (IException) is thrown. If writing or reading fails, an exception is thrown.

Parameters
command[]: Contains the command to be sent (size of 1, 2 or 4 bytes, depending on the command).
sizeCommand: Contains the size (in bytes) of the command (1, 2 or 4).
response: Array of the given size (sizeResponse) where the response of the micro-controller can be / is stored. If no return value is expected, the pointer can be NULL.
sizeResponse: Contains the size of the command (1 or 2, in case of no response is expected the value has to be 0). Thus, a sizeResponse value of 0 indicates that no response from the micro-controller will be read.
Returns
Returns true if writing / and reading was done successfully.

Implemented in SerialComLINUX.
